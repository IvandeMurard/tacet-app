Notes de conception · Itération 4

→ Pourquoi pas de polygones : les formes colorées masquent la carte, empêchent l'orientation et la projection (se repérer dans son futur quartier). La carte doit rester le héros — premium, hospitality-feel.
→ Pourquoi pas heatmap : les données IRIS sont discrètes (1 score / zone), pas continues. Un heatmap créerait des dégradés faux entre zones (interpolation entre score 80 et 30 = mensonge visuel).
→ Score dots = IQAir × Airbnb : pattern éprouvé (pin sur carte propre). Chaque dot = 1 donnée précise, aucune interpolation. L'utilisateur comprend déjà « cercle coloré = tap pour détails ».
→ Ambient Glow (OFF par défaut) : le défaut premium = carte propre. Le glow ajoute une atmosphère de couleur soft pour ceux qui le souhaitent. Toggle dans AppNav. Prépare aussi la couche futur social sentiment.
→ Carte : fond clair = #EDE8DF (crème chaud). Fond sombre = #0C1524 (navy profond). Rues et repères toujours visibles.
→ SearchBar : autofocus desktop uniquement. Mobile : pas d'autofocus — le clavier masquerait la carte.
