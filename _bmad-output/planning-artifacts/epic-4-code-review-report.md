# Epic 4 Code Review Report

**Epic:** Progressive Web App & Offline (Stories 4.1–4.4)
**Review type:** Adversarial code review
**Criteria:** Story acceptance criteria, architecture (Serwist precache, runtime caching, offline UX), NFRs, FR28–FR31.

---

## Summary

Epic 4 implements Serwist PWA support, offline caching, the OfflineBanner, PWAInstallPrompt, and manifest/icons. The manifest (4.4) and service worker infrastructure (4.1) are wired correctly. However, two AC-critical gaps exist: (a) the core GeoJSON data files (`paris-noise-iris.geojson`, `iris-centroids.geojson`) are never precached or runtime-cached, so the map cannot render offline despite story 4.1's explicit AC; (b) `lastVisitedZone` is loaded from `localStorage` into `MapContext` but never consumed — no component reads it to auto-display the last zone when the user is offline (story 4.2 AC: "Score and zone info are available"). Both components (`OfflineBanner`, `PWAInstallPrompt`) lack unit tests (NFR-M1). Three lower-severity findings: PWAInstallPrompt triggers immediately on `beforeinstallprompt` instead of after a meaningful action; the manifest has no maskable icon; `reloadOnOnline: true` destroys user state on reconnect.

## Findings

| ID | Severity | Story | File / Area | Finding | Recommendation |
|----|----------|-------|-------------|---------|----------------|
| **E4-H1** | High | 4.1 | `tacet/src/app/sw.ts`, `tacet/next.config.mjs` | AC: "paris-noise-iris.geojson is precached so the map can render offline." `next.config.mjs` provides no `additionalPrecacheEntries`; `sw.ts` uses only `defaultCache`. `@serwist/next` does not auto-include `public/data/` files — only build-output assets are injected into `__SW_MANIFEST`. Without these files in the cache, MapContainer's `fetch("/data/paris-noise-iris.geojson")` fails offline and the map is blank. `iris-centroids.geojson` (cluster dots) has the same problem. | Add runtime caching (CacheFirst or StaleWhileRevalidate) for `/data/*.geojson` in `sw.ts`, or add `additionalPrecacheEntries` in `next.config.mjs`. CacheFirst is appropriate since GeoJSON is static between deployments. Alternatively add both files to `additionalPrecacheEntries: [{ url: "/data/paris-noise-iris.geojson", revision: null }, { url: "/data/iris-centroids.geojson", revision: null }]` in the Serwist config. |
| **E4-M1** | Medium | 4.2 | `tacet/src/contexts/MapContext.tsx`, `tacet/src/app/MapPageClient.tsx` | AC: "the last visited zone's Score and zone info are available … when the user opens the app". `loadLastZoneFromStorage()` runs on init and populates `lastVisitedZone` in `MapContext`. However, `lastVisitedZone` is exposed in the context value but **never destructured or used in any component**. `MapPageClient` only renders `<IrisPopup>` when `selectedZone !== null`; it never reads `lastVisitedZone`. The OfflineBanner shows only the zone name, not the Score or tier. Story 4.2 AC ("Score and zone info are available") is not met. | In `MapPageClient`, initialise `selectedZone` from `lastVisitedZone` on mount if offline (or unconditionally, as the UX for "restore last zone"). A simple approach: in a `useEffect`, call `setSelectedZone(lastVisitedZone)` on first render if `lastVisitedZone !== null` and `selectedZone === null`. This auto-opens the IrisPopup with the cached Score and zone info on load. |
| **E4-M2** | Medium | 4.1 / 4.2 | `tacet/src/app/sw.ts` | Architecture spec: "RUMEUR (network-first, fallback cached)" in the SW runtime cache. `defaultCache` from `@serwist/next` does not add explicit strategies for Next.js API routes (`/api/rumeur`, `/api/chantiers`). The server-side 3-min in-memory cache handles freshness while online; however, if the network is completely unavailable, the SW has no cached RUMEUR response to serve, and the client receives a network error. Story 4.2 AC: "offline fallback for RUMEUR shows cached data with a 'données en cache' message" — not reliably met. | Add a NetworkFirst (with NetworkOnly fallback timeout) runtime cache entry in `sw.ts` for `/api/rumeur` and `/api/chantiers`. Suggested: `{ matcher: /\/api\/(rumeur|chantiers)/, handler: new NetworkFirst({ cacheName: "api-cache", networkTimeoutSeconds: 5, plugins: [...] }) }`. Since `RumeurStatusBar` already handles `fallback: true` gracefully, the SW just needs to serve the last cached response when offline. |
| **E4-M3** | Medium | 4.2 / 4.3 | `tacet/src/components/tacet/OfflineBanner.tsx`, `PWAInstallPrompt.tsx` | No unit tests for `OfflineBanner` or `PWAInstallPrompt`. NFR-M1: Vitest >= 80% coverage. `OfflineBanner` has testable logic: `online`/`offline` event listeners, `localStorage` read, fallback text. `PWAInstallPrompt` has testable logic: `sessionStorage` flag suppression, `beforeinstallprompt` deferral, `handleInstall`, `handleDismiss`. | Add `OfflineBanner.test.tsx` (show/hide on online/offline events; zone name from localStorage; fallback text when no zone) and `PWAInstallPrompt.test.tsx` (hidden when sessionStorage flag set; shown on event; Installer calls prompt(); Dismiss sets flag). |
| **E4-L1** | Low | 4.3 | `tacet/src/components/tacet/PWAInstallPrompt.tsx` | AC: "after the user has performed a meaningful action (e.g. first zone tap)". Current code sets `showPrompt(true)` immediately when `beforeinstallprompt` fires (which often happens within seconds of page load), before any user interaction. This violates the deferred-until-engagement UX pattern. | Hold the deferred event but delay showing the dialog. Options: (a) expose a `triggerInstallPrompt()` function from context (or a prop) and call it from `MapPageClient` after `selectedZone` changes from null to non-null; (b) add a minimum engagement flag (e.g. `sessionStorage` key set on first zone tap) before showing. Approach (a) is simplest: pass `deferredPrompt` down and call `setShowPrompt(true)` from `MapPageClient` on first zone tap. |
| **E4-L2** | Low | 4.4 | `tacet/public/manifest.json` | Lighthouse PWA audit: "Manifest does not have a maskable icon." Both 192 and 512 icons use `"purpose": "any"`. Browsers that support adaptive icons (Android, Chrome) expect at least one icon with `"purpose": "maskable"` for correct home-screen display. | Add a maskable variant: either add `{ "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable" }` as a second entry for the 512 icon, or create a dedicated `icon-512-maskable.png` with safe-zone padding and use that. Simplest fix: add `"purpose": "any maskable"` to the 512 entry if the icon fills the entire canvas without important content in the corners. |
| **E4-L3** | Low | 4.1 | `tacet/next.config.mjs` | `reloadOnOnline: true` forces a full page reload when the device reconnects. If the user is viewing an IrisPopup, comparing zones, or has an open RUMEUR popup at that moment, all transient state is destroyed. The architecture does not require automatic reload on reconnect — `RumeurStatusBar` already handles freshness indication, and RUMEUR/Chantiers data refreshes naturally on the next SWR interval. | Set `reloadOnOnline: false`. The `RumeurStatusBar` polling (via SWR) will resume automatically when online and update the data display without disrupting the user. |

## Resolved / Already correct

- **Service worker infrastructure (4.1):** Serwist properly wired — `@serwist/next` wraps `next.config.mjs`, `sw.ts` at `src/app/sw.ts`, `skipWaiting: true`, `clientsClaim: true`, `navigationPreload: true`. SW is disabled in development (`disable: process.env.NODE_ENV === "development"`). ✅
- **App shell precaching (4.1):** `__SW_MANIFEST` injects all Next.js build artifacts (pages, JS chunks, CSS, fonts). App shell repeat-visit performance (FR31) is met. ✅
- **Manifest completeness (4.4):** `name`, `short_name`, `start_url: "/"`, `display: "standalone"`, `lang: "fr"`, `theme_color`, `background_color` — all required fields present. ✅
- **Icons 192 + 512 (4.4):** Both sizes present in `/public/icons/`. `apple-touch-icon.png` (180px) declared in `layout.tsx`. ✅
- **OfflineBanner online/offline detection (4.3):** Listens to `window.online`/`offline` events, initialises from `navigator.onLine`, hydration-safe `typeof window === "undefined"` guard. Non-blocking amber banner design. ✅
- **PWAInstallPrompt deduplication (4.3):** `sessionStorage` flag prevents re-showing in the same session after dismiss or accept. ✅
- **LastZone persistence (4.2):** `MapContext` saves `IrisProperties` to `localStorage` on every zone select, reads on init. Storage key consistent with `OfflineBanner` (`"tacet-last-zone"`). `IrisProperties.name` matches `zone.name` read in `OfflineBanner`. ✅
- **`lastVisitedZone` in context (4.2):** Exposed in `MapContextValue` interface and `useMemo` value. The value is available for any component that needs it — the gap is only that no component currently consumes it. ✅ (data layer) / ❌ (display layer — see E4-M1).
